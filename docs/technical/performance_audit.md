### BOTTLENECKS IDENTIFIED

#### 1. Render-Blocking Resources (CSS/Fonts) — Impact: HIGH
*   **Location**: `css/style.css` and Google Fonts imports.
*   **Evidence**: Both are linked in the `<head>`, delaying First Contentful Paint (FCP).
*   **Root cause**: Standard CSS linking without critical path separation.
*   **Fix**: 
    *   Identify "Above-the-Fold" CSS and inline it in `<style>` tags.
    *   Load the remaining `style.css` asynchronously using `rel="print" onload="this.media='all'"` pattern.
    *   Self-host fonts or use `font-display: swap` to prevent FOIT (Flash of Invisible Text).

#### 2. Redundant JSON Fetching — Impact: MED
*   **Location**: `js/app.js` -> `renderExamList()`
*   **Evidence**: Every subject page navigation (e.g., `math.html`, `science.html`) re-fetches the entire `subjects.json`.
*   **Root cause**: Monolithic metadata file and lack of client-side caching.
*   **Fix**: 
    *   Implement **Lazy Loading Strategy**: Split `subjects.json` into subject-specific manifests (e.g., `meta_math.json`, `meta_science.json`).
    *   Only fetch the specific manifest required for the active view.
    *   Use `localStorage` to cache metadata with a TTL (Time-To-Live).

#### 3. JSON Parsing Latency (Scalability) — Impact: LOW (Current) / MED (Future)
*   **Location**: `js/quiz.js` -> `init()`
*   **Evidence**: 20-question JSON files are fetched and parsed on the main thread.
*   **Root cause**: Synchronous parsing of larger JSON strings can block the main thread, especially on low-end mobile devices.
*   **Fix**: 
    *   Ensure JSON structure is optimized (flat vs. nested).
    *   For larger sets (>50 questions), implement chunked parsing or use a Web Worker to handle data processing.

#### 4. Subject Icon Optimization — Impact: MED
*   **Location**: `app.js` -> `subjectMeta`
*   **Status**: Currently using Emojis (efficient but limited branding).
*   **Optimization Fix**:
    *   Transition to **SVGs** for icons: Inline critical icons to eliminate HTTP requests.
    *   If using Raster images (PNG/JPG): Convert to **WebP/AVIF** formats.
    *   Implement `loading="lazy"` for subject cards below the fold on the dashboard.

### LAZY LOADING STRATEGY FOR DATA
To optimize the loading of non-active subject data:
1.  **Route-Based Partitioning**:
    *   Move from `data/subjects.json` (all subjects) to `data/subjects/[id].json`.
    *   `index.html` only loads a lightweight directory file (name, icon, id).
    *   `math.html` only loads `data/subjects/math.json`.
2.  **Resource Preloading**:
    *   Use `<link rel="prefetch">` for the next likely subject JSON when a user hovers over a subject card on the dashboard.
3.  **On-Demand Question Loading**:
    *   Currently, the quiz loads all 20 questions at once. For larger banks, implement a "Next 5" loading strategy to reduce initial payload.
